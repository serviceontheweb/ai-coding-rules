# Philosophy

AI coding rules should behave like engineering guardrails, not process theater.

## Principles

- Small instructions are more useful than exhaustive manuals.
- Repository facts matter more than generic best practices.
- Safety rules should be explicit and hard to weaken by accident.
- Validation claims require evidence.
- Memory should preserve durable lessons, not every session detail.
- Public examples should be reusable and free of private operational context.

## Non-Goals

This project is not:

- an agent runtime;
- a deployment system;
- a compliance product;
- a marketing framework;
- a replacement for code review or tests.

It is a lightweight starting point for consistent AI-assisted engineering.
