# AGENTS.md Standards

`AGENTS.md` is the portable operating contract for coding agents.

## Required Sections

- Session start rules.
- Operating principles.
- Safety and approval requirements.
- Context discipline.
- Validation expectations.
- Git rules.
- Project-specific facts.

## Writing Rules

- Keep root instructions concise.
- Use direct engineering language.
- Name real commands only.
- Put durable lessons in `LESSONS.md`.
- Put detailed technical reference material in the wiki.
- Do not include credentials, private paths, or proprietary workflows.

## Project-Specific Additions

Add only facts the agent needs:

- stack and package manager;
- test, lint, build, and formatting commands;
- deployment constraints;
- local safety constraints;
- known non-obvious gotchas.
