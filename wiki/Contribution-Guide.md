# Contribution Guide

Contributions should improve clarity, portability, and safety without expanding
the framework into a large manual.

## Standards

- Keep README concise.
- Put technical reference content in the wiki.
- Keep examples generic and public-safe.
- Prefer links over duplicated explanations.
- Use direct engineering language.
- Do not weaken approval gates or secret handling.

## Where Content Belongs

- Shared rules: `core/`
- Workflows: `skills/`
- Machine-readable examples: `policies/`
- Durable knowledge templates: `memory/`
- Tool setup: `adapters/`
- Technical reference: `wiki/`

## Validation

Before proposing changes:

- run Markdown formatting checks when possible;
- check internal links;
- search for non-public details;
- confirm examples are generic.
