# Workflow Patterns

Workflow patterns describe how agents should move through common engineering
tasks.

## Standard Session

1. Read `AGENTS.md`.
2. Read relevant lessons and changelog entries.
3. Inspect the smallest useful context.
4. Make a scoped change.
5. Run validation matched to risk.
6. Report changed files, validation, and remaining risks.

## Review Workflow

1. Inspect the diff and nearby code.
2. Prioritize defects, regressions, missing tests, and safety issues.
3. Reference exact files and lines.
4. Separate confirmed findings from questions.

## Closeout Workflow

1. Check repository status.
2. Review the diff.
3. Update memory only for durable lessons.
4. Update changelog only for completed user-visible work.
5. Report validation honestly.
