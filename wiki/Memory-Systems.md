# Memory Systems

Memory files preserve durable project knowledge for future agent sessions.

## Files

- `LESSONS.md` records reusable lessons and gotchas.
- `CHANGELOG.md` records completed user-visible work.
- Architecture decision records capture long-term design choices.
- Anti-pattern records document practices the project avoids.

## Add a Lesson When

- the same mistake could recur;
- the project has a non-obvious constraint;
- a command, dependency, or config has a gotcha;
- the user corrects a durable preference;
- a workflow needs a permanent warning.

## Do Not Add

- secrets;
- guesses;
- temporary scratch notes;
- one-off task summaries;
- private operational details.
