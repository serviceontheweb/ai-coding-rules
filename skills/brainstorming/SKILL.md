# Brainstorming

## Purpose

Explore useful options without treating speculation as implementation.

## Triggers

- Architecture or design planning.
- Naming or structure options.
- Tradeoff analysis.
- Roadmap or migration planning.
- "What should we do?" questions.

## Checklist

- Separate facts from assumptions.
- Keep options practical and comparable.
- Prefer existing project constraints over generic patterns.
- Identify risks and validation needs.
- Avoid claiming implementation status.

## Execution Pattern

1. Restate the decision to be made.
2. List two to four viable options.
3. Compare tradeoffs briefly.
4. Recommend one path with reasoning.
5. Name the next concrete action.

## Escalation Conditions

- The decision affects production, data, security, legal, or irreversible work.
- Required facts are unavailable and guessing would be risky.
- The user asks to proceed with an option that conflicts with safety rules.
